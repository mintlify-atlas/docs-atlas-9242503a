## Overview

Splat provides comprehensive terminal integration through tmux sessions, enabling AI-powered debugging in interactive terminal environments. The system includes session management, process monitoring, and ZMQ-based communication.

## ZapShell

An interactive command-line shell built with Python's `cmd` module that provides the main interface for Splat.

### Initialization

```python
from cli.cli import ZapShell

shell = ZapShell()
shell.cmdloop()
```

The shell initializes with:
- `intro`: Welcome message
- `prompt`: `zap> `
- `ctx`: Click context (set after initialization)
- `zapper`: Zapper instance for ZMQ communication
- `term_sesh`: TermSesh instance for tmux session management

### start

Start a new tmux session with process monitoring enabled.

```bash
zap> start
tmux session and monitoring started successfully
```

**What it does:**
1. Creates a `Zapper` instance for ZMQ communication
2. Creates a `TermSesh` with session name `zapper_session`
3. Opens a new terminal window attached to the tmux session
4. Starts a `ProcessMonitor` thread to track terminal output
5. Sends initial commands to the terminal (`ls`, `cat setup.py`)

<Info>
The start command automatically opens a new terminal window on macOS (using Terminal.app) or Linux (using gnome-terminal).
</Info>

## TermSesh Methods

### open_new_terminal()

Creates a new tmux session and opens it in a terminal window.

```python
if term_sesh.open_new_terminal():
    print("Terminal session started")
    term_sesh.send_to_terminal("ls -la")
```

**Platform Support:**
- **macOS**: Uses AppleScript to open Terminal.app
- **Linux**: Uses gnome-terminal
- **Windows**: Not yet supported

**Returns:** `bool` - True if successful, False otherwise

<Note>
The method automatically kills any existing session with the same name before creating a new one.
</Note>

### read_tmux_output()

Reads and cleans output from the tmux session.

```python
output = term_sesh.read_tmux_output()
if output:
    print(f"New output: {output}")
```

**Features:**
- Captures the current tmux pane content
- Cleans ANSI escape sequences and Unicode characters
- Returns only new output (diff from previous read)
- Maintains output history in `tmux_stack`

**Returns:** `str` - New output since last read, or empty string

**Source:** `~/workspace/source/cli/term_sesh.py:90-103`

---

### clean_tmux_output(raw_output: str)

Cleans and formats raw tmux output by removing ANSI codes and duplicates.

```python
cleaned = term_sesh.clean_tmux_output(raw_tmux_output)
```

**Cleaning operations:**
1. Removes ANSI escape sequences (colors, formatting)
2. Removes Unicode prompt characters (powerline symbols)
3. Strips whitespace from lines
4. Removes empty lines
5. Removes duplicate consecutive lines

**Returns:** `str` - Cleaned output with one line per unique entry

Starts the background monitoring thread.


- Creates a daemon thread running `monitor_tmux()`
- Thread automatically terminates when main program exits

**Source:** `~/workspace/source/cli/process_monitor.py:11-16`

---

### monitor_tmux()

Background loop that reads tmux output and publishes via ZMQ.

```python
def monitor_tmux(self):
    while self.is_running:
        output = self.term_sesh.read_tmux_output()
        if output:
            self.term_sesh.publisher.send_json({
                'type': 'tmux_output',
                'data': output,
                'timestamp': time.time()
            })
        time.sleep(1)
```

**Published Message Format:**
```json
{
    "type": "tmux_output",
    "data": "command output here",
    "timestamp": 1234567890.123
}
```

<Info>
The monitor checks for new output every 1 second. Adjust `time.sleep(1)` to change polling frequency.
</Info>

**Source:** `~/workspace/source/cli/process_monitor.py:17-26`

## Complete Usage Example

Here's a complete example showing how all components work together:

```python
from cli.cli import ZapShell
from cli.term_sesh import TermSesh
from cli.process_monitor import ProcessMonitor
from cli.zap import Zapper

# Initialize the Zapper for ZMQ communication
zapper = Zapper(port=5555)
zapper.start()

# Create a terminal session
term_sesh = TermSesh(
    port=5555,
    session_name='my_debug_session'
)

# Open a new terminal window with tmux
if term_sesh.open_new_terminal():
    print("Terminal opened successfully")
    
    # Start monitoring the terminal output
    monitor = ProcessMonitor(term_sesh)
    monitor.start_monitoring()
    
    # Send commands to the terminal
    term_sesh.send_to_terminal("python -m pytest tests/")
    
    # Wait and read output
    import time
    time.sleep(2)
    output = term_sesh.read_tmux_output()
    print(f"Test output: {output}")
    
    # Send code for analysis
    term_sesh.send_code_segment({
        'file_path': 'tests/test_example.py',
        'code': open('tests/test_example.py').read(),
        'metadata': {'type': 'test'},
        'action': 'debug'
    })
    
    # Cleanup when done
    monitor.is_running = False
    term_sesh.kill_tmux_session()
    zapper.stop()
```

## ZMQ Message Types

The terminal integration publishes several message types over ZMQ:

### tmux_output

Published by ProcessMonitor when new terminal output is detected.

```json
{
    "type": "tmux_output",
    "data": "test output...",
    "timestamp": 1234567890.123
}
```

### info

Published by TermSesh for informational messages.

```json
{
    "type": "info",
    "data": "tmux session started"
}
```

### code_segment

Published when code is sent for analysis (compressed).

```json
{
    "file_path": "src/main.py",
    "code": "eJwLycgsVgCiRIWS...",
    "metadata": {},
    "action": "analyze"
}
```

## Environment Requirements

<Warning>
Splat's terminal integration requires tmux to be installed on your system.
</Warning>

**Installation:**

```bash
# macOS
brew install tmux

# Ubuntu/Debian
sudo apt-get install tmux

# Fedora
sudo dnf install tmux
```

**Python Dependencies:**
- `click` - CLI framework
- `zmq` (pyzmq) - ZeroMQ messaging
- `cmd` - Built-in Python module
