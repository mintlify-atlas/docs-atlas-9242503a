## Get Your First Debug Response

This guide will walk you through using Splat to debug your first Python error. You'll learn the basic commands and see how Splat's AI-powered analysis works in practice.

<Note>
Before starting, make sure you've completed the [installation](/installation) steps and set up your Groq API key.
</Note>

## Prerequisites Checklist

<Steps>
  <Step title="Verify Splat is installed">
    ```bash
    splat --help
    ```
    
    You should see the Splat help menu with available commands.
  </Step>
  
  <Step title="Confirm Groq API key is set">
    ```bash

    ```
    
    This should display your API key. If empty, set it:
    ```bash
    export GROQ_API_KEY="your-api-key-here"
    ```
  </Step>
  
  <Step title="Navigate to your Python project">
    ```bash
    cd /path/to/your/python/project
    ```
  </Step>
</Steps>

## Basic Usage

### The `splat squash` Command

Splat uses the `squash` command to debug your Python code. The basic syntax is:

```bash
splat squash "python3 your_script.py"
```

Simply wrap your normal Python execution command in quotes and prepend it with `splat squash`.

<Info>
Use the exact command you would normally use to run your script, including any arguments or flags.
</Info>

## Your First Debug Session

### Example 1: Simple Error

Let's say you have a script `foo.py` that's throwing an error:

```bash
splat squash "python3 foo.py"
```

**What happens:**

1. Splat runs `python3 foo.py`
2. Captures any error output
3. Analyzes the stack trace
4. Gathers context from related files
5. Sends to Groq AI for analysis
6. Returns a detailed debug response

**Example output:**

### Example 2: With Arguments

If your script takes command-line arguments:

```bash
splat squash "python3 main.py --input data.csv --verbose"
```

Splat will pass all arguments through to your script while monitoring for errors.

## Using Relational Mode (`-r` flag)

The `-r` flag enables **relational context gathering**, which provides deeper analysis by including all files related to your error stack.

### When to Use `-r`

Use the `-r` flag when:
- Your error involves multiple files or modules
- You need deeper contextual analysis
- The error is complex or spans different parts of your codebase
- You want the most accurate debugging response

### Example: Relational Mode

```bash
splat squash -r "python3 main.py"
```

**Difference from basic mode:**

<CodeGroup>

```bash Basic Mode
splat squash "python3 main.py"
# Analyzes only the files in the immediate stack trace
```

```bash Relational Mode (-r)
splat squash -r "python3 main.py"
# Analyzes stack trace files + all related modules, imports, and dependencies
```

</CodeGroup>

<Warning>
Relational mode (`-r`) takes slightly longer but provides significantly more context. Use it for complex errors or when the basic mode doesn't provide enough insight.
</Warning>

## Understanding Splat's Output

Splat provides several types of information in its debug response:

### 1. Error Detection
```
Error detected: TypeError: unsupported operand type(s)
```
The specific error that was caught.

### 2. Stack Trace
```
Stack trace:
  File "main.py", line 42, in process_data
  File "utils.py", line 18, in transform
```
Where the error occurred in your code.

### 3. AI Analysis
Detailed explanation of:
- What caused the error
- Why it happened
- How to fix it
- How to prevent it in the future

### 4. Code Suggestions
Specific code changes or fixes when applicable.

## Common Use Cases

<AccordionGroup>
  <Accordion title="Debugging Import Errors">
    ```bash
    splat squash "python3 app.py"
    ```
    Splat will identify missing imports and suggest the correct import statements.
  </Accordion>
  
  <Accordion title="Fixing Type Errors">
    ```bash
    splat squash -r "python3 calculator.py"
    ```
    Use `-r` to understand type mismatches across multiple functions.
  </Accordion>
  
  <Accordion title="Resolving Syntax Errors">
    ```bash
    splat squash "python3 script.py"
    ```
    Get instant feedback on syntax issues with suggestions for fixes.
  </Accordion>
  
  <Accordion title="Debugging Scripts with Dependencies">
    ```bash
    splat squash -r "python3 main.py --config prod.yaml"
    ```
    Relational mode helps track errors through complex dependency chains.
  </Accordion>
</AccordionGroup>

## Known Limitations

<Warning>
Be aware of these current limitations:
- Splat will not conform to formatting configurations when inserting code
- May have issues with sub-module entrypoints not originating in the root directory
</Warning>

## Example Workflow

Here's a complete workflow example:

```bash
# 1. Navigate to your project
cd ~/projects/my-python-app

# 2. Run Splat on your script
splat squash "python3 main.py"

# 3. Review the AI analysis

# 4. Apply suggested fixes to your code

# 5. Run again to verify the fix
splat squash "python3 main.py"

# 6. If needed, use relational mode for deeper analysis
splat squash -r "python3 main.py"
```

<Info>
Splat works globally across all your Python projects without any per-project configuration. Just use it whenever you need instant debugging help!
</Info>
